# Research: Book Master Plan

**Feature**: Book Master Plan
**Date**: 2025-12-06

## 1. Node.js Version
- **Decision**: Use Node.js v18 (LTS) or v20 (LTS).
- **Rationale**: Docusaurus v3 requires Node.js >= 18.0.
- **Alternatives**: Node 16 (deprecated), Node 21 (non-LTS).

## 2. Sidebar Structure
- **Decision**: Use `sidebars.js` with explicit category definitions for the 5 modules.
- **Rationale**: Explicit control allows ordering weeks chronologically rather than alphabetically. Autogenerated sidebars might mix up "Week 1" and "Week 10".
- **Alternatives**: Autogenerated sidebars (simpler but less control).

## 3. GitHub Pages Deployment
- **Decision**: Use the standard `deploy` command or GitHub Actions workflow provided by Docusaurus.
- **Rationale**: `GIT_USER=<GITHUB_USERNAME> npm run deploy` is the simplest start, but a GitHub Action is preferred for the Constitution's "Continuous Delivery" principle.
- **Implementation**: Use the `facebook/docusaurus/actions/deploy` pattern.

## 4. Branding & Styling
- **Decision**: Use standard Docusaurus `classic` preset but override colors in `src/css/custom.css` to match "Physical AI" theme (e.g., cool blues, robotics/metallic accents).
- **Rationale**: Low effort, high impact. Full custom theme is out of scope for MVP.

## 5. MDX Components
- **Decision**: Enable MDX support (default in v3).
- **Rationale**: Required for interactive elements specified in functional requirements.
